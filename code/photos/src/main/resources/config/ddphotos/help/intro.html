Welcome to the DD Photos

DD Photos helps you quickly create, manage, and publish photo self-hosted photo galleries. It uses the open-source ddphotos project under the hood, which itself uses Docker.

Here is the typical workflow from first setup to a published site:


  1. Pick a home for your DD Photos site. Choose a folder where all DD Photos files will live - a good default is ~/my-ddphotos in your home folder.

  2. Create a starter site. Run ddphotos init to create the DD Photos folders with a sample configuration so you can explore commands right away.

  3. Edit Site Details. Set your site name, description, public URL, and other settings on the Config tab.

  4. Organize Your Photos. The key to working with DD Photos is understanding than an album is essentially all photos in a given folder. Organize your photos into specific folders like Patagonia Hiking, Hannah's Graduation, or Mom's Birthday. Many users put these in their Pictures folder. Other users might use a Dropbox or Google Drive folder.

  5. Define Albums. Add album entries that point to the folders containing your photos. Albums support nested folders, cover images, and custom descriptions.

  6. Run photogen. Resizes photos for fast downloading and builds the data index files that the site needs. Run this whenever you add or change photos or edit site or album details. Resized photos are saved in the albums folder under your DD Photos site folder.

  7. Run run. Starts a live-reload development server at http://localhost:5173 so you can preview the site in a browser. After editing your site or album details, you need to re-run photogen, and the development server automatically picks up the changes.

  8. Run build. Generates the final static site output in the build folder of your DD Photos site folder.

  9. Run serve. Serves the static output locally via Apache at http://localhost:8000 for a final check before publishing.

  10. Deploy. Publish your site. Quick options include exporting and uploading to Cloudflare Pages (free, unlimited bandwidth) via wrangler, or Surge.sh (free) via surge. Advanced options include deploying via rsync or S3 as configured in a config/site.env file.
