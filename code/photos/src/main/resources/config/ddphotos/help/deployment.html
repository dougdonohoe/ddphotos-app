Deploying Your Site

Once you are happy with your site preview, deploying means publishing it to the internet so anyone can visit it. DD Photos first creates a self-contained copy of your finished site (the build step), then uploads it to the hosting service of your choice.

The two easiest options are Cloudflare Pages and Surge - both are free and work well for photo galleries. More advanced options (your own server via rsync, or Amazon S3) are described further down.


Cloudflare Pages (recommended)

Cloudflare Pages is a free hosting service with unlimited bandwidth, which makes it a great fit for photo sites. Your site will be published at an address like https://my-unique-site.pages.dev.

  1. Export your site. Run export with the --cloudflare option so the special files Cloudflare needs are included automatically. This creates an export folder inside your DD Photos folder.

  2. Sign in (first time only). DD Photos runs wrangler login for you, which opens a web page for you to log in to and connect your free Cloudflare account. Your login is remembered for next time.

  3. Publish. Run wrangler to upload the exported site. The very first upload creates the project for you and assigns your .pages.dev address.

Surge

Surge is another simple, free option. Your site will be published at an address like https://my-unique-site.surge.sh.

  1. Export your site. Surge needs real copies of every file rather than shortcuts, so run export with the --copy option.

  2. Publish. Run surge to upload the exported site. The first time you run, you'll be directed to run a command in Terminal so you can create your account and/or login. Your login is remembered after that.

Advanced: your own server (rsync) or Amazon S3

DD Photos can also publish directly to your own web server (via rsync) or to Amazon S3 with CloudFront. These options require a one-time setup: create a text file named site.env inside your Config folder containing your server or bucket details. Once that file exists, you publish by running the deploy command, which figures out which method to use.

For publishing to your own server with rsync, site.env looks like: 

    RSYNC_HOST=user@your-server.example.com
    RSYNC_DEST=/path/to/your/web/root/
    CLOUDFRONT_ID=YOUR_CLOUDFRONT_DISTRIBUTION_ID   # optional

For publishing to Amazon S3 (with CloudFront), site.env looks like: 

    S3_BUCKET=your-s3-bucket
    CLOUDFRONT_ID=YOUR_CLOUDFRONT_DISTRIBUTION_ID   # optional

These advanced methods also need valid credentials (an SSH key for rsync, or AWS credentials for S3). See the official guide for the complete details.
